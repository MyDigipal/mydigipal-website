Critère Statut Détail

Marketing checkbox par défaut Coché par défaut Le RGPD exige un opt-in explicite pour le marketing - la checkbox ne devrait PAS être cochée par défaut
Lien "Gérer les cookies" dans le footer Absent window.openCookieSettings() est exposé mais aucun lien dans le footer ne l'appelle. L'utilisateur ne peut pas modifier ses préférences après la première visite.
Non-conformité RGPD : L'absence de lien pour gérer les préférences de cookies dans le footer est une violation du RGPD. Le règlement exige que l'utilisateur puisse modifier son consentement à tout moment, de manière aussi simple qu'il l'a donné.
Approche hybride : L'analytics_storage est granted par défaut sous "intérêt légitime". Cette approche est défendue par certaines interprétations (notamment UK ICO), mais contestée par d'autres DPA européennes (CNIL France, BfDI Allemagne). MyDigipal étant une entreprise UK, c'est acceptable post-Brexit, mais à surveiller si le public cible inclut l'UE.

Architecture technique du tracking

Flux de données :
1. BaseLayout.astro : Initialise dataLayer + Consent Mode V2 (avant GTM)
2. BaseLayout.astro : Charge GTM de manière différée (requestIdleCallback / interaction)
3. CookieConsent.astro : Gère le consentement, met à jour gtag consent + pousse consent_update
4. Calculator.tsx : Pousse calculator_form_submit et calculator_mode_selected
5. contact.astro : Pousse contact_form_submit
6. BaseLayout.astro : Listener global pour phone_click et email_click
7. GTM : Reçoit les événements, vérifie le consentement via triggers, fire les tags GA4/Meta/Hotjar
